ChatGPT Auto-Project Suggestion

As AI chats accumulate, conversations become fragmented and users eventually stop organizing them. This project introduces an auto-project suggestion system that groups related chats and helps users continue their work without losing context.

01

Problem

As users interact with ChatGPT over time, their chat history grows increasingly fragmented. Three core pain points emerged:

The Restart Loop — The cost of finding past conversations grows so high that users simply open a new chat and start over, losing all prior context.

Re-Prompting Tax — Every new chat requires re-explaining files, background, and goals that were already established in a previous session.

Organization Fatigue — As chats accumulate, manually sorting and categorizing them becomes an overwhelming task users abandon entirely.

"Finding my old chats takes too long, so I just open a new window and start over."

02

AI Prototyping Approach

Two core solution concepts were explored and prototyped:

Auto Project Suggestions — Analyzes recurring themes across recent chat history and proactively suggests grouping them into a named project.

Continuity Suggestions — When a new chat is similar to an existing project, the system suggests either continuing the previous conversation or adding the new chat to that project.

3D assets were also designed to communicate key states: an Empty Folder placeholder shown when a project has no chats yet, and a Grouping Chats loading animation shown while the system scans recent chats for relevant conversations.

03

Design Direction

The solution introduces a Projects layer into ChatGPT's navigation, allowing users to manage related conversations as a single, continuous workflow.

Projects Page — A dedicated Projects section in the sidebar lets users see all ongoing project folders alongside recent chats. Users can create new projects or navigate into existing ones.

Projects Suggestion — On the Projects page, ChatGPT surfaces AI-generated project suggestions based on recurring themes in recent chat history (e.g. "UX Research Interview", "English Study"). Users can accept a suggestion with one tap to create the project instantly.

Quick Projects Creation — Accepting a suggestion immediately creates the project and populates it — reducing friction and helping users organize their context naturally without deliberate effort.

Add Chats to the Project — Inside a project, users can select relevant past chats from a modal checklist and add them to the project, unifying scattered conversations and files into one coherent thread.

From Recent Chats — A "Find relevant chats" button scans existing chat history and surfaces related conversations to add to the current project — consolidating context automatically.

04

Outcome

The AI-prototyped feature demonstrated that proactive organization — surfaced at the right moment — reduces the cognitive load of managing growing chat histories. By meeting users where they are (in a new chat or on the Projects page), the system makes organization feel effortless rather than effortful.
